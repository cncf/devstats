# THE RUNBOOK — shared `affiliations` DB migration (final, signed-off version)

Status: implementation complete and dirty in all 4 repos (devstats 301 modified + 12 new, devstatscode 5 modified,
devstats-docker-images 2 modified + 1 new, devstats-helm 5 modified + 1 new). `go build ./...` and all `bash -n` pass.
Legacy mode (GHA2DB_AFFILIATIONS_DB unset) is byte-identical behavior everywhere.

The shared affs_lock: `gha_locks(name primary key, owner, dt)` in the `devstats` DB. `devel/lock_shared.sh <db> <name> <owner> [attempts]`
acquires atomically (`INSERT ... ON CONFLICT DO NOTHING` + owner verification, 720x10s retries then fails visibly);
`devel/unlock_shared.sh` releases only its own lock (`WHERE name AND owner`). There is NO automatic stale-lock stealing
(affs TSDB regens may legitimately hold the lock for many hours - calc_metric allows 24h). If a holder pod is killed hard
and the lock is abandoned, waiters fail after ~2h with the owner token printed; clear manually after verifying no affs
job is running:
  psql devstats -c "delete from gha_locks where name = 'affs_lock'"
Used by affs.sh (shared-mode branch), import_affs_shared.sh, proj_affs_bootstrap.sh; legacy flag paths untouched.

SEED — consolidate from the namespace's project DBs (read-only on sources, fail-closed):
```bash
kubectl $CTX -n $NS cp $LIST $PRIMARY:/tmp/dbs.txt -c devstats-postgres
kubectl $CTX exec -itn $NS $PRIMARY -c devstats-postgres -- bash
set -euo pipefail
psql -q -v ON_ERROR_STOP=1 -U gha_admin affiliations -c "create unlogged table staging_actors (like gha_actors); create unlogged table staging_emails (like gha_actors_emails); create unlogged table staging_names (like gha_actors_names)"
for db in $(cat /tmp/dbs.txt); do
  kind=$(psql -tAc "select coalesce((select relkind::text from pg_class c join pg_namespace n on n.oid=c.relnamespace where n.nspname='public' and c.relname='gha_actors'),'-')" "$db")
  case "$kind" in
    r) ;;
    f) echo "SKIP $db (already shared)"; continue ;;
    *) echo "ABORT: $db gha_actors relkind='$kind'"; exit 1 ;;
  esac
  echo "=== $db"
  psql -q -v ON_ERROR_STOP=1 "$db" -c "\copy (select * from gha_actors) to stdout" | psql -q -v ON_ERROR_STOP=1 -U gha_admin affiliations -c "\copy staging_actors from stdin"
  psql -q -v ON_ERROR_STOP=1 "$db" -c "\copy (select * from gha_actors_emails where origin = 1) to stdout" | psql -q -v ON_ERROR_STOP=1 -U gha_admin affiliations -c "\copy staging_emails from stdin"
  psql -q -v ON_ERROR_STOP=1 "$db" -c "\copy (select * from gha_actors_names where origin = 1) to stdout" | psql -q -v ON_ERROR_STOP=1 -U gha_admin affiliations -c "\copy staging_names from stdin"
done
psql -v ON_ERROR_STOP=1 -U gha_admin affiliations <<'SQL'
insert into gha_actors(id, login, name, country_id, sex, sex_prob, tz, tz_offset, country_name, age)
select id, login, name, country_id, sex, sex_prob, tz, tz_offset, country_name, age
from (
  select s.*, row_number() over (
    partition by id, login
    order by ((name is not null and name <> '')::int + (country_id is not null and country_id <> '')::int
      + (sex is not null)::int + (sex_prob is not null)::int + (tz is not null and tz <> '')::int
      + (tz_offset is not null)::int + (country_name is not null)::int + (age is not null)::int) desc
  ) rn from staging_actors s
) x where rn = 1
on conflict (id, login) do nothing;
insert into gha_actors_emails(actor_id, email, origin)
select actor_id, email, max(origin) from staging_emails group by actor_id, email
on conflict (actor_id, email) do update set origin = greatest(gha_actors_emails.origin, excluded.origin);
insert into gha_actors_names(actor_id, name, origin)
select actor_id, name, max(origin) from staging_names group by actor_id, name
on conflict (actor_id, name) do update set origin = greatest(gha_actors_names.origin, excluded.origin);
drop table staging_actors, staging_emails, staging_names;
analyze gha_actors, gha_actors_emails, gha_actors_names, gha_actors_affiliations, gha_companies, gha_countries, gha_imported_shas, gha_bot_logins;
SQL
exit
```
SEED is safe to re-run anytime (used again as the "delta"). It skips flipped DBs, preserves every (id,login) pair
including negative IDs, and merges enrichment origins.

SWITCHOVER — the full-namespace cutover:
```bash
# 1. suspend everything (incl. affiliations-import, excl. backups) and drain
kubectl $CTX -n $NS get cronjobs -o name | grep -v backup | xargs -n1 -I{} kubectl $CTX -n $NS patch {} --type merge -p '{"spec":{"suspend":true}}'
while true; do
  act=$(kubectl $CTX -n $NS get jobs -o jsonpath='{range .items[?(@.status.active>0)]}{.metadata.name}{"\n"}{end}')
  [ -z "$act" ] && break; echo "waiting: $act"; sleep 30
done
# 2. delta: rerun SEED   3. RECONCILE   4. rerun SEED (enrichment re-sweep) + MAPS
# 5. re-copy + sha256-verify the 3 SQL files on the CURRENT primary (re-run PREAMBLE first)
# 6. flip loop (fail-closed):
set -euo pipefail
for db in $(cat $LIST); do
  kind=$($K psql -tAc "select coalesce((select relkind::text from pg_class c join pg_namespace n on n.oid=c.relnamespace where n.nspname='public' and c.relname='gha_actors'),'-')" "$db")
  if [ "$kind" = "f" ]; then
    n=$($K psql -tAc "select count(*) from information_schema.foreign_tables where foreign_table_schema='public' and foreign_server_name='affiliations'" "$db")
    m=$($K psql -tAc "select count(*) from pg_user_mappings where srvname='affiliations'" "$db")
    if [ "$n" = "11" ] && [ "$m" = "4" ]; then echo "OK (already migrated): $db"; continue; fi
    echo "ABORT: $db partially migrated (ft=$n maps=$m)"; exit 1
  elif [ "$kind" != "r" ]; then echo "ABORT: $db relkind='$kind'"; exit 1; fi
  echo "=== $db"; # FDW(db); FLIP(db)
done
# 7. upgrade every CronJob-bearing project release with the new chart + cutover values,
#    preserving each release's own values. Discover the owning releases via the CJ annotations
#    (only type=cron / affiliations-cron CJs; excludes backups/velero/affs-import automatically):
kubectl $CTX -n "$NS" get cronjobs -o json | jq -r '.items[]
  | select((.metadata.labels.type == "cron") or (.metadata.labels.type == "affiliations-cron"))
  | .metadata.annotations["meta.helm.sh/release-name"] // "<no-owner>"' | sort -u
#    then upgrade each: explicit skipAffiliationsImport=1 so no ordinary release ever renders
#    the central importer (do NOT use --reuse-values):
for rel in <the releases from the list above>; do
  helm -n "$NS" get values "$rel" -o yaml > "/tmp/vals-$rel.yaml"
  helm -n "$NS" upgrade "$rel" ./devstats-helm -f "/tmp/vals-$rel.yaml" \
    --set affiliationsDB=affiliations --set checkImportedSHA='' --set affiliationsGetAffsFiles='' \
    --set skipAffiliationsImport=1
done
#    (add --set affsFdwUsePassword=1 only if the password FDW variant was chosen at the pilot)
# 8. explicitly unsuspend ALL cronjobs - helm's three-way merge does NOT remove the
#    kubectl-added spec.suspend, because none of the CJ templates ever rendered that field:
kubectl $CTX -n "$NS" get cronjobs -o name | xargs -n1 -I{} kubectl $CTX -n "$NS" patch {} --type merge -p '{"spec":{"suspend":false}}'
```

   sanity (count(*), EXPLAIN with Remote SQL: `explain select * from gha_actors where login = 'xyz';`)
